观察者模式：在对象之间定义一对多的依赖，这样一来，当一个对象改变状态，依赖它的对象都会收到通知，并自动更新。

示例：气象站会获取数据，有更新时会推送给不同的布告板显示不同的内容。

分析：
1.已经提供一个WeatherData类，能从物理设备获取数据，提供get方法。
2.如果数据改变时，提供一个方法，逐一更改每个布告板的状态，则有新布告板添加时，要手动修改代码，可扩展性不高。
3.为了达到松耦合的效果，应该提供主题接口，让WeatherData类实现它。接口中提供列表储存注册的观察者，提供添加、移除观察者方法，并在数据
变化时，通知注册了的观察者。
4.提供观察者接口，每个布告板要实现它，通过多态便于扩展。另外，提供一个接口，规范布告板的行为。
